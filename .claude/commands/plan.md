## planコマンド

ユーザーからタスクの説明を受け取り、plan.mdに追記する。

### 手順

1. **タスク確認**: ユーザーのタスク説明を確認
2. **不明点の洗い出し**: 以下を確認
   - タスクの目的・ゴール
   - 成果物（ファイル、機能など）
   - 依存関係（他のタスクとの関連）
   - 技術的な不明点
3. **質問**: 不明点があればユーザーに確認
4. **承認取得**: タスク内容をまとめて承認を求める
5. **plan.md更新**: 承認後、「次のステップ（予定）」にタスクを追記

### タスク記載形式

```markdown
## 次のステップ（予定）

- [タスク名]: [簡潔な説明]
  - 目的: [なぜ必要か]
  - 成果物: [何ができるか]
```

### 注意事項

- 1つのタスクは具体的かつ実行可能な単位にする
- 曖昧なタスクは分割または明確化する
- 既存タスクとの重複を避ける
